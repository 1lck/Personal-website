在学校做项目时，最常被问到的问题就是“Spring Boot 项目到底怎么部署最好”。结合自己的校园实践与实习经验，我把方案分成三层：最轻量的 Jar 直跑、容器化部署、以及基于 Spring Cloud 的多服务发布。下面是我的笔记。

## 方案一：Jar + Systemd

- **适用**：个人项目、课程实验。
- **做法**：使用 Maven/Gradle 打包可执行 Jar，服务器只需安装 OpenJDK。通过 Systemd 管理启动、停止与日志。
- **亮点**：维护成本低，随时可以 SSH 登录查看日志。搭配 Nginx 做反向代理即可应付大多数 Demo。

## 方案二：Docker 化

- **适用**：需要快速扩展和回滚的团队项目。
- **步骤**：
  1. 编写多阶段 Dockerfile，第一阶段构建，第二阶段运行；
  2. 使用 `docker-compose` 或 K8s 将应用、Redis、数据库等服务组合在一起；
  3. 通过环境变量配置数据库、消息队列等连接信息。
- **优势**：版本可追溯、跨平台一致，配合 Harbor 或阿里云镜像仓库就能实现 CI/CD。

## 方案三：Spring Cloud + K8s

- **适用**：像“趣游兔”这样的多租户系统，涉及网关、配置中心、限流等环节。
- **关键点**：
  - 使用 Nacos/Consul 管理配置和服务注册；
  - Gateway + Sentinel 完成路由及熔断限流；
  - 通过 K8s Deployment 滚动升级，配合 HPA 实现按负载自动扩容。
- **经验**：配置中心一定要和 Git 绑定，避免线上修改后无法回滚；日志必须集中到 ELK/Prometheus，否则排障很痛苦。

## 我的建议

1. **场景优先**：个人练手不要一开始就上 K8s，简单方案更有利于理解原理。
2. **参数配置**：无论哪种方案，敏感信息都放在环境变量或密钥管理服务里。
3. **监控 + 备份**：哪怕是 Jar 模式，也要至少做好日志切割和数据库备份。

希望这份笔记能帮你快速选出适合自己的部署方式。如果你有更好的实践，欢迎联系我一起交流。
