小程序是旅游业务的主战场，重构后我们仍遇到重复提交、登录态过期、大文件上传失败等痛点。我负责梳理关键链路并提升可用性，最终将成功率拉到 99.3%+。以下是核心思路。

## 订单幂等与熔断

- 使用业务 ID + Redis 时间窗实现幂等校验，避免重复扣款。
- 接入 Sentinel 定义熔断策略，在支付网关异常时自动降级到排队模式。
- 通过消息队列异步写入流水，确保即便回滚也能回查。

## 分片上传与续传

- 对 200MB 以上的素材上传，调用 OSS 分片接口，客户端记录进度，服务器端校验 MD5。
- 结合 Spring Boot WebFlux 实现断点续传，失败自动重新请求当前分片。
- 成果：大文件失败率从 12% 降到 1% 以内。

## 登录态续期

- 在 Sa-Token 基础上设计刷新 token 机制，前端在 token 即将过期时主动续期。
- Redis 维护用户 Session，并设置“滑动过期”，避免频繁重新登录。

## 监控与演练

- 建立链路追踪日志，实时观测下单、上传、支付、票据查询等环节。
- 每周进行压测与故障演练，验证重试策略，Prometheus + Grafana 提供实时告警。

通过这次复盘，我意识到“细节即体验”。这些实践也被我迁移到学校的竞赛报名系统，效果同样显著。
